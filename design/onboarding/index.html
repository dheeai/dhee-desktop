🔌 Exposing ComfyUI for programmatic access

  1. The running ComfyUI app is the API — there's no separate server to start. Same machine? Use http://127.0.0.1:8188.
  2. ComfyUI on another machine / GPU box? Relaunch it with --listen 0.0.0.0 and use http://<that-host>:8188. Add --enable-cors-header '*' if the browser blocks it.
  3. Connection refused = ComfyUI isn't running or is bound to localhost only. Reachable but errors on /object_info = update ComfyUI.
