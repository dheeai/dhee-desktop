Problem 1 — coalesce consecutive Dhee messages

One byline per author run

A single multi-step turn used to render as N stamped boxes, each with its own DHEE eyebrow. Now a run of consecutive Dhee messages is one authored block — byline once, an amber spine down the side, messages flowing beneath. The eyebrow only returns when the author flips.

Problem 2 — intentional treatment per content kind

Each kind reads as what it is

The chat mixes conversation, working notes, run events and artifacts. Instead of flattening everything into stamped bubbles, each gets a distinct, restrained treatment.

Authored proseFinal answer & streaming text — Instrument Sans body under the one byline. This is the conversation.
Tool I/OEvery dhee_* call + its result is a first-class card (status readout, cascade preview, take-strip, picker…) — call is the header, result is the body. Never folded, because we know what each returns.
Run eventsPipeline notices (“Run resumed”, “Run started”) are centered amber slugs — clearly outside the conversation, never a bubble.
ArtifactsScene / shot outputs render as filmstrip stills with sprocket edges and a mono caption — an artifact, not a paragraph.
Live activityThe pinned ON-SET transport above the composer — the only thing that animates, so motion always means “Dhee is working.”
